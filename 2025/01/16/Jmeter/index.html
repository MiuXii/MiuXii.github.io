Jmeter

Created2025-01-16|Updated2025-05-28|性能测试接口测试
|Post Views:

简介

前置条件

基于java语言开发，需要JDK环境，解压完Jmeter且配置好环境变量后就可以启动使用了。

启动方法：

  • 配置好jdk系统变量后，可以在bin目录下打开ApacheJMeter.jar，就可以启动jmeter
  • bin目录下打开jmeter.bat也可以启动jmeter，但是有个控制台，控制台不能关闭，关闭控制台会把jmeter也关闭了

文件目录详解

  • backups:自动备份目录
  • bin：含jmeter.bat启动文件，jmeter.properties核心配置文件（语言、分布式压测、持续集成报告生成等）
  • docs:文档
  • extras:存放和第三方集成的构建文件
  • lib:存放jar包

命令行用法（非GUI、报告生成、持续集成）

常用参数,例如jmeter -n -t test.jmx -l result.jtl -e -o results -f
-n：命令行模式运行JMeter
-t：指定要执行的jmx文件
-l：指定测试结果的日志文件(类似于查看结果树)
-j：指定JMeter日志文件(INFO、ERROR、DEBUG等)
-g：指定结果文件的路径(指定jtl文件生成报告)
-e：在测试结束后生成HTML报告
-o：指定用于生成HTML报告的目标目录
-f：强制将JMeter进程结束，或者在运行非GUI测试时，立即结束测试并生成结果文件

分布式压测参数,例如#jmeter -i 192.168.1.101
-r：远程运行JMeter客户端和服务器模式 jmeter-server -Djava.rmi.server.hostname=192.168.1.100
-s：运行JMeter作为服务器(用于分布式测试) jmeter-server
-c：指定远程服务器的 IP 地址或主机名 jmeter -c 192.168.1.100
-i：指定本地 IP 地址，用于远程服务器通信

其他参数
-H：设置用于代理服务器的主机名。jmeter -H proxy.example.com
-P：设置用于代理服务器的端口号. jmeter -P 8080

组件介绍

  • 测试计划：一个完整的测试容器，包含多个线程组，对不同接口的不同请求
  • 配置元件：接口配置（请求头、默认请求、cookie、csv数据文件配置等）
    • HTTP请求默认值：设置后续线程组中默认的协议类型、主域名（生产环境、测试环境）、端口号等， 后续取样器中路径只需要从后面开始填写
    • HTTP信息头管理器：
      • 一般请求数据是json的话需要添加Content-Type，application/json;charset=UTF-8，否则发送请求失败。
    • HTTP Cookie管理器:在访问指定域名服务器时，自动存储cookie且自动在同一站点传递cookie(第一次请求一个IP时，如果返回cookie则自动存储，后续再访问该IP及子域名时则自动带上)
    • JDBC connection COnfiguration :数据库连接配置，需要放到上边
  • 前置处理器：请求前的操作（脚本动态数据生成等）
  • 后置处理器：请求后的操作（JSON提取器、正则表达式提取、接口关联等）
    • 接口关联-jsonpath写法(键值对)：
      • 自定义一个变量名:提取完后续请求要用到这一变量用${变量名}来引用
      • $根节点（大括号），.取子结点的值，子节点可以往下顺延，例子：$.key.access_token
      • 没取到就是default默认
    • 接口关联-正则表达式写法(文本)：
      • 引用名称:自定义变量名
      • 左边界字符串(.*?)右边界字符串，例如name="key" value="(.*?)"
      • 模板:$1$默认取表达式匹配到的第一个参数
      • 匹配数字：表示从第几组开始取
      • 没取到就是default默认
  • 线程组：设定1-N个线程
    • 逻辑控制器：条件成立才请求接口、循环请求等
  • 定时器：延迟请求
  • 取样器：
    • http请求设置：请求方式、请求路径、请求参数、请求头等
    • 调试取样器：查看请求字节数据和中间变量
  • 断言：判断请求是否成功，失败则爆红打印错误结果
    • 响应断言（是否包含指定信息）：1.测试模式中添加匹配的文本、字符串等; 2.自定义错误消息
    • json断言(判断状态码等信息是否匹配)：1.添加要匹配的键的路径json path,例如’$.state’; 2.添加expected value预期的值
  • 监听器：收集测试结果
  • 测试片段：和线程组同级，不能单独运行，必须放到线程组里的控制器一起使用（include、模块控制器），作用是模块化管理测试用例，线程组只能选中部分导出，这个可以直接吧一个模块的导出
    • 模块控制器：自己测试的模块，启用后可以保存导出
    • include控制器：leader把多人的模块拿过来一起测试

    AES，DES，Base64

不可以解密的
MD5,SHA系列，HmacSHA系列

连接数据库

  • 选择数据库对应版本的jar包, Mysql官网下载链接

  • 将jar包复制到JMeter所在文件夹的lib/ext路径下，重启JMeter

Variable Name for cerated pool：自定义用户名，后续使用数据库时作为标识。
Validation Query：校验连接是否成功，mysql选择 select 1
Datebase URL：数据库的URL，jdbc:mysql://url:端口/数据库名?serverTimezone=UTC&characterEncoding=utf-8
JDBC Driver class： 数据库驱动 mysql选择commysql.jdbc.Driver
Username：数据库用户名
Password：数据库用户名对应密码

  • 选择线程组后右键添加取样器 JDBC Request
  • 查询出数据后，可以利用beanshell断言打印到控制台或添加全局变量

性能压测

线程组参数设置

  • 线程数：设置虚拟用户数。一个虚拟用户占用一个进程或线程。线程数就相当于虚拟用户数。

  • Ramp-Up：设置的线程数启动时长，单位为秒。如果线程数为100，准备时长为20秒，那么需要20秒启动100个线程，平均每秒启动5个线程。

  • 循环次数：每个线程发送请求的个数。如果线程数为100，循环次数为2，那么每个线程发送2次请求，总请求数为100*2=200次。如果勾选了“永远”复选框，那么所有线程会循环发送请求，直到手动单工具栏停止按钮，或者设置的线程运行时间结束才会停止运行。

性能指标

  • 并发数:不是线程数（并发用户数），是同一时刻下访问呢服务器的连接数
  • 吞吐量:
    • QPS：接口每秒处理的查询次数
    • TPS：每秒处理的业务流程数
      • 金融行业：1000TPS~50000TPS，不包括互联网化的活动
      • 保险行业：100TPS~100000TPS，不包括互联网化的活动
      • 制造行业：10TPS~5000TPS
      • 互联网电子商务：10000TPS~1000000TPS
      • 互联网中型网站：1000TPS~50000TPS
      • 互联网小型网站：500TPS~10000TPS
  • 响应时间:从客户端发起请求到接收到服务端返回的响应所耗费的时间
    • 90%响应时间：所有用户的响应时间进行排序，第 90%的响应时间。
  • 资源利用率:不超过80%
  • 事务成功率:成功率不低于99.9%
